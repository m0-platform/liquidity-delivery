# M0 Limit Order Protocol (EVM)

This folder contains the Ethereum Virtual Machine (EVM) implementation of the M0 Limit Order Protocol. It uses Foundry for building and testing the contracts. The primary contract is `src/OrderBook.sol`. It depends on M0's Portal V2 architecture for sending and receiving crosschain messages related to order fills.

## Development

### Installation

You may have to install the following tools to use this repository:

- [Foundry](https://github.com/foundry-rs/foundry) to compile and test contracts

### Compile

Run the following command to compile the contracts:

```bash
make build
```

### Test

To run all tests:

```bash
make tests
```

Run test that matches a test contract:

```bash
forge test --mc <test-contract-name>
```

Test a specific test case:

```bash
forge test --mt <test-case-name>
```

### Code quality

[Prettier](https://prettier.io) is used to format code. Use it by running:

```bash
pnpm run prettier
```

[Solhint](https://protofire.github.io/solhint/) is used to lint Solidity files. Run it with:

```bash
pnpm run solhint
```

To fix solhint errors, run:

```bash
pnpm run solhint-fix
```

### Documentation

The documentation can be generated by running:

```bash
pnpm run doc
```

It will run a server on port 4000, you can then access the documentation by opening [http://localhost:4000](http://localhost:4000).
